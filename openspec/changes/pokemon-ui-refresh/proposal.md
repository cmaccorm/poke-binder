## Why

Poke-binder works functionally, but the current interface lacks the visual polish expected of a Pokémon-themed app. A focused UI refresh can make the product feel more cohesive, readable, and delightful without changing binder management behavior.

## What Changes

- Refresh the binder UI with a Pokémon-inspired visual system using a Poké Ball palette, nicer surfaces, and stronger visual hierarchy.
- Improve the presentation of binder and card imagery so the refreshed layout stays crisp and stable across sizes.
- Preserve existing binder workflows while elevating the look and feel of the shelf, binder, and edit surfaces.

## Capabilities

### New Capabilities
- `pokemon-ui-theme`: Defines the refreshed visual language, including palette, surfaces, elevation, spacing, and polish expectations for the app shell and binder views.
- `catalog-image-optimization`: Defines image loading and presentation requirements so card art remains sharp, responsive, and visually stable in the refreshed UI.

### Modified Capabilities
- `digital-pokemon-binder`: Updates binder-facing requirements to align with the refreshed visual treatment while preserving existing binder behavior and layouts.

## Impact

UI components, theme tokens, layout styling, surface treatment, card image presentation, and any supporting image-loading behavior. No API or data-model changes are expected.
