## ADDED Requirements

### Requirement: Pokémon-inspired app theme
The application MUST present binder screens with a Poké Ball-inspired palette, layered surfaces, and consistent spacing that makes the interface feel polished and cohesive.

#### Scenario: Binder view uses themed surfaces
- **WHEN** the user opens the shelf or a binder
- **THEN** the page uses the shared Pokémon-inspired palette and surface treatment
- **AND** text and controls remain legible against those surfaces

### Requirement: Visual hierarchy supports navigation
The application MUST use clear visual hierarchy for binder names, page content, and editing affordances so users can distinguish primary actions from decorative elements.

#### Scenario: User scans the binder view
- **WHEN** the binder view is displayed
- **THEN** the active page, controls, and editable slots are visually distinct
- **AND** decorative styling does not obscure binder content
