## Why

The app currently uses SQLite (file-based), which works for local development but doesn't support concurrent connections, scaling, or cloud deployment patterns. Switching to a cloud-hosted PostgreSQL (Supabase) provides a production-ready database with proper connection management, reliability, and the ability to scale.

## What Changes

- Change Prisma datasource from `sqlite` to `postgresql` in `prisma/schema.prisma`
- Update `prisma/migrations/migration_lock.toml` to use `postgresql` provider
- Update `DATABASE_URL` in `.env` to a Supabase PostgreSQL connection string
- Add `pg` (node-postgres) as a dependency in `package.json`
- Regenerate Prisma migrations for PostgreSQL dialect via `prisma migrate dev`
- Regenerate Prisma client via `prisma generate`
- Delete the local SQLite database file (`prisma/dev.db`)

## Capabilities

### New Capabilities
- `cloud-database`: Database connection and configuration for cloud-hosted PostgreSQL via Supabase

### Modified Capabilities
- `digital-pokemon-binder`: Database backend changes from SQLite to PostgreSQL (connection string, provider, dependencies)

## Impact

- **Configuration files**: `prisma/schema.prisma`, `prisma/migrations/migration_lock.toml`, `.env`
- **Dependencies**: `package.json` — add `pg`
- **Generated files**: `src/generated/prisma/` — regenerated by `prisma generate`
- **Database file**: `prisma/dev.db` — removed
- **Application code**: No changes needed — Prisma abstracts the SQL dialect, all queries are dialect-agnostic
- **Migrations**: Must be regenerated via `prisma migrate dev` against the Supabase instance
